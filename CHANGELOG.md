## Guidelines for Contributors

When adding changes to this file:

1. Add changes under the `[Unreleased]` section
2. Use the categories: Added, Changed, Fixed, Deprecated, Removed, Security
3. Keep entries brief but descriptive
4. Link to issues/PRs when relevant
5. Format links as `[#123](https://github.com/your-username/repo/issues/123)`

Example entry:

```markdown
### Added

- New feature description ([#123](https://github.com/your-username/repo/pull/123))
```
