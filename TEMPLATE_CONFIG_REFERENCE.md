# Universal OSS Template - Configuration Summary

This document provides a quick reference for all configuration files in the template.

## üìã File Inventory

### Root Configuration Files

| File                   | Purpose                                         |
| ---------------------- | ----------------------------------------------- |
| `package.json`         | Project metadata, dependencies, and npm scripts |
| `.eslintrc.json`       | JavaScript/TypeScript linting rules             |
| `.prettierrc`          | Code formatting configuration                   |
| `commitlint.config.js` | Conventional commit enforcement rules           |
| `.editorconfig`        | Unified editor settings for all file types      |
| `.gitignore`           | Git ignore patterns                             |
| `.env.example`         | Environment variables template                  |

### Documentation Files

| File                 | Purpose                                              |
| -------------------- | ---------------------------------------------------- |
| `README.md`          | Main project documentation and quick start           |
| `CONTRIBUTING.md`    | Contribution guidelines and workflow                 |
| `CODE_OF_CONDUCT.md` | Community standards and expectations                 |
| `SECURITY.md`        | Security policy and vulnerability reporting          |
| `CHANGELOG.md`       | Version history (auto-generated by Semantic Release) |
| `LICENSE`            | MIT License                                          |

### GitHub Configuration

| Directory/File                              | Purpose                                     |
| ------------------------------------------- | ------------------------------------------- |
| `.github/workflows/ci.yml`                  | CI pipeline (lint, format, test on PR/push) |
| `.github/workflows/release.yml`             | Automated semantic release on main merge    |
| `.github/ISSUE_TEMPLATE/bug_report.md`      | Bug report issue template                   |
| `.github/ISSUE_TEMPLATE/feature_request.md` | Feature request issue template              |
| `.github/PULL_REQUEST_TEMPLATE.md`          | PR submission template                      |

### Git Hooks

| File                | Purpose                                   |
| ------------------- | ----------------------------------------- |
| `.husky/commit-msg` | Validates commit messages with commitlint |

### Documentation

| Directory/File         | Purpose                                   |
| ---------------------- | ----------------------------------------- |
| `docs/architecture.md` | System design and structure documentation |
| `examples/example.md`  | Usage examples and patterns               |

## üöÄ GitHub Actions Workflows

### CI Pipeline (`ci.yml`)

**Triggers:** Push and Pull Request

**Jobs:**

- Lint: Runs ESLint
- Format Check: Verifies Prettier formatting

### Release Pipeline (`release.yml`)

**Triggers:** Push to main branch

**Jobs:**

- Semantic Release: Automatic version bumping and changelog generation

## üìù Conventional Commits Reference

**Format:**

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Types:**

- `feat` - New feature (minor bump)
- `fix` - Bug fix (patch bump)
- `docs` - Documentation
- `style` - Code formatting
- `refactor` - Code refactoring
- `perf` - Performance improvement
- `test` - Tests
- `chore` - Dependencies/tooling
- `ci` - CI/CD configuration

**Breaking Change:** Add `BREAKING CHANGE:` footer for major bump

## üîê Security Best Practices

3. Use dependabot for dependency updates

## üîÑ Release Workflow

1. Create feature branch: `git checkout -b feat/feature-name`
2. Make changes with conventional commits
3. Create pull request
4. Get approval
5. Merge to `main`
6. Semantic Release automatically:
   - Analyzes commits
   - Determines version (major/minor/patch)
   - Updates CHANGELOG.md
   - Creates Git tag
   - Publishes GitHub Release
